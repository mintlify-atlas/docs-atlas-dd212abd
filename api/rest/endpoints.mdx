This page documents all available REST API endpoints for the IOTA blockchain.

## Node Information

### GET /

Get node information including software version and chain ID.

**Stability:** Stable

```bash cURL
curl https://api.iota.io/api/v1/
```

**Response:**

```json
{
  "name": "IOTA Node API",
  "version": "1.0.0",
  "chainId": "4btiuiMPvEENsttpZC7CZ53DruC3MAgfznDbASZ7DR6S"
}
```

### GET /health

Health check endpoint.

**Stability:** Stable

```bash cURL
curl https://api.iota.io/api/v1/health
```

**Response:**

```json
{
  "status": "ok"
}
```

## Checkpoints

### GET /checkpoints

List checkpoints with pagination.

**Stability:** Stable

<ParamField query="limit" type="u32" default="50">
  Page size limit (max: 100)
</ParamField>

<ParamField query="start" type="u64">
  Starting checkpoint sequence number. Defaults to latest if not specified.
</ParamField>

<ParamField query="direction" type="Direction" default="descending">
  Pagination direction: `ascending` or `descending`
</ParamField>

<ParamField query="contents" type="boolean" default="false">
  Include checkpoint contents
</ParamField>

```bash cURL
curl "https://api.iota.io/api/v1/checkpoints?limit=10&direction=ascending"
```

**Response:**

<ResponseField name="checkpoint" type="CheckpointSummary" required>
  Checkpoint summary data
  
  <Expandable title="CheckpointSummary">
    <ResponseField name="sequenceNumber" type="u64">
      Checkpoint sequence number
    </ResponseField>
    <ResponseField name="digest" type="String">
      Base58-encoded checkpoint digest
    </ResponseField>
    <ResponseField name="timestamp" type="DateTime">
      Timestamp when checkpoint was created
    </ResponseField>
    <ResponseField name="networkTotalTransactions" type="u64">
      Total transactions in network up to this checkpoint
    </ResponseField>
    <ResponseField name="previousCheckpointDigest" type="String">
      Digest of previous checkpoint
    </ResponseField>
    <ResponseField name="epochId" type="u64">
      Epoch ID
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="signature" type="ValidatorAggregatedSignature" required>
  Aggregated validator signatures
</ResponseField>

<ResponseField name="contents" type="CheckpointContents">
  Checkpoint contents (if requested)
</ResponseField>

**Headers:**
- `X-Iota-Cursor`: Cursor for next page

### GET /checkpoints/{checkpoint}

Get a specific checkpoint by sequence number or digest.

**Stability:** Stable

<ParamField path="checkpoint" type="CheckpointId" required>
  Checkpoint sequence number (e.g., "1000") or digest (Base58-encoded)
</ParamField>

<ParamField query="contents" type="boolean" default="false">
  Include checkpoint contents
</ParamField>

```bash cURL
curl "https://api.iota.io/api/v1/checkpoints/1000?contents=true"
```

**Status Codes:**
- `200 OK` - Checkpoint found
- `404 Not Found` - Checkpoint doesn't exist
- `410 Gone` - Checkpoint has been pruned

### GET /checkpoints/{checkpoint}/full

Get full checkpoint data including all transactions, effects, events, and objects.

**Stability:** Unstable

**Content-Type:** Only supports `application/bcs`

<ParamField path="checkpoint" type="CheckpointId" required>
  Checkpoint sequence number or digest
</ParamField>

```bash cURL
curl -H "Accept: application/bcs" \
  https://api.iota.io/api/v1/checkpoints/1000/full \
  --output checkpoint_1000.bcs
```

**Status Codes:**
- `200 OK` - Full checkpoint data returned (BCS format)
- `400 Bad Request` - Invalid accept type
- `404 Not Found` - Checkpoint doesn't exist
- `410 Gone` - Checkpoint objects have been pruned

### GET /checkpoints/full

List full checkpoints with all associated data.

**Stability:** Unstable

## Transactions

### GET /transactions/{transaction}

Get a transaction by its digest.

**Stability:** Unstable

<ParamField path="transaction" type="Digest" required>
  Base58-encoded transaction digest
</ParamField>

```bash cURL
curl https://api.iota.io/api/v1/transactions/Dv5XRBjWvZTwiHVrhUetvDTgxyM8xVMb6P8pCAZv51tq
```

**Response:**

<ResponseField name="digest" type="Digest" required>
  Transaction digest
</ResponseField>

<ResponseField name="transaction" type="Transaction" required>

</ResponseField>

<ResponseField name="signatures" type="Vec<UserSignature>" required>
  User signatures
</ResponseField>

<ResponseField name="effects" type="TransactionEffects" required>
  Transaction effects
</ResponseField>

<ResponseField name="events" type="TransactionEvents">
  Events emitted by the transaction
</ResponseField>

<ResponseField name="checkpoint" type="u64">

</ResponseField>

</ResponseField>

### GET /transactions

List transactions with pagination.

**Stability:** Unstable

<ParamField query="limit" type="u32" default="50">
  Page size (max: 100)
</ParamField>

<ParamField query="start" type="String">
  Transaction cursor in format `{checkpoint}.{index}`. Defaults to latest.
</ParamField>

<ParamField query="direction" type="Direction" default="descending">
  Pagination direction
</ParamField>

```bash cURL
curl "https://api.iota.io/api/v1/transactions?limit=20"
```

**Headers:**
- `X-Iota-Cursor`: Transaction cursor for next page (format: `{checkpoint}.{index}`)

### POST /transactions

Execute a signed transaction.

**Stability:** Unstable

<ParamField body="txBytes" type="Base64" required>
  BCS-encoded transaction bytes
</ParamField>

<ParamField body="signatures" type="Vec<Base64>" required>
  Transaction signatures
</ParamField>

<ParamField query="effects" type="boolean" default="true">
  Wait for and return transaction effects
</ParamField>

<ParamField query="events" type="boolean" default="false">
  Include transaction events
</ParamField>

<ParamField query="finality" type="EffectsFinality" default="certified">
  Finality level: `certified` or `executed`
</ParamField>

```bash cURL
curl -X POST https://api.iota.io/api/v1/transactions \
  -H "Content-Type: application/json" \
  -d '{
    "txBytes": "AAACACCNu...",
    "signatures": ["ANjGTW3..."]
  }'
```

**Response:** Same as GET /transactions/{transaction}

### POST /transactions/simulate

Simulate transaction execution without committing.

**Stability:** Unstable

<ParamField body="txBytes" type="Base64" required>
  BCS-encoded transaction bytes
</ParamField>

<ParamField query="effects" type="boolean" default="true">
  Include transaction effects
</ParamField>

<ParamField query="events" type="boolean" default="false">
  Include transaction events
</ParamField>

```bash cURL
curl -X POST "https://api.iota.io/api/v1/transactions/simulate?effects=true" \
  -H "Content-Type: application/json" \
  -d '{"txBytes": "AAACACCNu..."}'
```

### POST /transactions/resolve

Resolve unresolved transaction data into a complete transaction.

**Stability:** Unstable

```bash cURL
curl -X POST https://api.iota.io/api/v1/transactions/resolve \
  -H "Content-Type: application/json" \
  -d '{...}'
```

## Response Headers

All responses include these headers:

- `X-Iota-Chain-Id` - Chain identifier
- `X-Iota-Software-Version` - Node software version
- `X-Iota-Cursor` - Pagination cursor (for paginated endpoints)
- `Content-Type` - Response content type (JSON or BCS)

## Error Responses

Error responses include a JSON object with an error message:

```json
{
  "error": "Description of the error"
}
```

Common HTTP status codes:

- `400 Bad Request` - Invalid request parameters
- `404 Not Found` - Resource not found
- `410 Gone` - Resource has been pruned from the node
- `500 Internal Server Error` - Server error
